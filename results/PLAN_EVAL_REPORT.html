Top Gap 1

PRD-018 critical

Overlay saved user data on every show appearance

This gap matters because search, AI mentions, and recommendations can show a title without the user's real status/rating/tag context, making discovery surfaces feel inconsistent and untrustworthy.

Top Gap 2

PRD-026 critical

Re-add preserves My Data and refreshes public data

This gap matters because the current delete-first plan can permanently discard the user's prior relationship data, so removing and later re-adding a show risks losing curation work.

Top Gap 3

PRD-091 important

Validate discovery with rubric and hard-fail integrity

This gap matters because the app's differentiator is taste-aware AI, and without explicit validation criteria the implementation can ship outputs that are formally parseable but off-brand or low-integrity.

Top Gap 4

PRD-033 important

Sync libraries/settings consistently and merge duplicates

This gap matters because cross-device or repeated-save flows can drift into inconsistent settings or duplicate library state unless merge behavior is defined beyond per-row show timestamps.

Top Gap 5

PRD-084 important

Deliver surprising but defensible taste-aligned recommendations

This gap matters because the recommendation engine could collapse into safe, obvious picks, which weakens the product's core promise of discovery that feels insightful rather than generic.

Risk Assessment

If executed as-is, the app would likely look complete and mostly work, but QA and early users would notice subtle trust breaks first: saved shows would not always carry the same personal state across discovery surfaces, removing and later re-adding a show could lose prior curation, and AI recommendations could be mechanically valid while drifting toward generic or insufficiently defended picks.

Remediation Guidance

The plan needs a small number of high-leverage additions: explicit cross-surface overlay rules for all show render paths, a defined re-add/tombstone strategy or alternate removal model, explicit session reset semantics for Ask and Alchemy, sync and duplicate-resolution rules for libraries and settings, and a rubric-backed discovery validation section with hard-fail checks for real-show integrity and on-brand recommendation quality.

Remaining Partial Or Missing Requirements

These are the requirements stakeholders would still be carrying forward as delivery risk if implementation started now.

PRD-018 critical partial

Overlay saved user data on every show appearance

The plan marks or resolves saved items, but it does not explicitly merge full status/tags/rating/Scoop overlays everywhere a show can appear.

PRD-026 critical partial

Re-add preserves My Data and refreshes public data

Hard-deleting the row clears My Data, but the plan never defines a strategy that would let a later re-add restore prior user data while refreshing catalog fields.

PRD-091 important missing

Validate discovery with rubric and hard-fail integrity

The plan does not add any rubric-based discovery validation or hard-fail acceptance checks to prove outputs meet the quality bar and real-show integrity standard.

PRD-004 important partial

Ignore `.env*` secrets except example

The ignore rules cover common `.env` files but do not clearly exclude `.env*` secrets generically while preserving only `.env.example`.

PRD-028 important partial

Use timestamps for sorting, sync, freshness

Timestamps drive conflict resolution and Scoop freshness, but no timestamp-driven sorting behavior is planned.

PRD-030 important partial

Keep Ask and Alchemy state session-only

The flows are described as session-retained, but the plan does not explicitly clear chat/history/results when the user leaves or resets those surfaces.

PRD-032 important partial

Show collection and rating tile indicators

Tile badges are fully specified on Home, but Search and other discovery tiles only call out in-collection marking, not rating indicators.

PRD-033 important partial

Sync libraries/settings consistently and merge duplicates

The plan covers per-field merge for shows but does not specify duplicate detection or end-to-end conflict handling for synced settings/library duplicates.

PRD-064 important partial

Keep primary actions early and page not overwhelming

The section order is deliberate, but the plan does not turn the “powerful but not overwhelming” principle into concrete anti-clutter layout rules.

PRD-074 important partial

Redirect Ask back into TV/movie domain

The prompt constrains Ask to TV/movies, but it does not describe the user-visible redirect or handoff behavior for out-of-domain requests.

PRD-078 important partial

Require concept selection and guide ingredient picking

The plan requires concept selection, but it omits the guidance copy that helps users choose the “ingredients” they want more of.

PRD-084 important partial

Deliver surprising but defensible taste-aligned recommendations

The recommendation contract covers concept citation and real-title resolution, but it does not define the “surprising but defensible” quality bar.
